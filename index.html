BCIT COMP 2132 수업을 위해 개발한 브라우저 기반 주사위 게임입니다. DOM 조작, 이벤트 기반 로직, 실시간 UI 업데이트 등 핵심 JavaScript 개념을 실습 형식으로 적용하기 위해 제작되었습니다.

프론트엔드JavaScript, HTML, CSS
도구Git, VS Code
  • 확률 기반 결과를 갖는 주사위 무작위화

  • 점수 추적 및 라운드 관리 로직
  • 게임 리셋 및 승리 조건 감지
문제 해결 & 배운 점

문제 해결

초기 과제는 여러 번의 롤에 걸쳐 점수를 잃지 않고 게임 상태를 관리하는 것이었습니다. 상태를 하나의 JavaScript 객체에 중앙화하고 그 단일 진실 소스에서 UI를 업데이트하는 방식으로 해결했습니다.

배운 점

이벤트 기반 프로그래밍과 DOM이 상태 변화에 반응하는 방식을 깊이 이해하게 되었습니다. 게임 로직과 표시 로직을 분리하여 코드를 디버그하고 확장하기 쉽게 만드는 법을 배웠습니다.

향후 개선 사항

  • 턴 기반 흐름을 갖춘 멀티플레이어 지원 추가
  • CSS 트랜지션으로 주사위 롤 애니메이션 적용
  • localStorage를 사용한 최고 점수 저장

💼 Immersio — 인턴십 프로젝트

비공개 / NDA

Immersio 인턴십 기간 중 개발한 AI 기반 언어 학습 채팅 플랫폼입니다. OpenAI API를 Django 백엔드 및 비동기 메시지 처리를 위한 Kafka와 통합하여 실시간 대화형 코칭을 제공하도록 설계되었습니다.

백엔드Python, Django
APIOpenAI API
메시징Kafka
도구Git, Postman
  • 문맥 인식 대화 응답을 위한 OpenAI API 통합
  • 비동기 논블로킹 채팅 흐름을 위한 Kafka 메시지 큐
  • 세션 및 사용자 컨텍스트를 관리하는 Django REST 백엔드
  • 학습자 수준에 맞춘 언어 학습 프롬프트
문제 해결 & 배운 점

문제 해결

초기에 OpenAI에 대한 동기 API 호출이 채팅 UI에서 눈에 띄는 지연을 일으켰습니다. Kafka를 도입하여 요청-응답 사이클을 분리하는 것이 해결책이었습니다. 프론트엔드가 메시지를 보내면 Kafka가 AI 응답을 비동기적으로 전달하여 UI의 응답성을 유지합니다.

배운 점

서드파티 AI API를 프로덕션 Django 애플리케이션에 통합하는 방법과, 메시지 큐가 채팅 기반 시스템에서 실시간 응답성 문제를 해결하는 방법을 배웠습니다. 실제 엔터프라이즈 소프트웨어에서 사용되는 비동기 아키텍처 패턴을 경험했습니다.

향후 개선 사항

  • 사용자 학습 진도 추적 및 어휘 기록 추가
  • 다국어 및 숙련도 수준 지원
  • 진정한 실시간 전달을 위한 WebSocket 구현

대출 관리 시스템

💼 Cypress Hills Partners — 인턴십 프로젝트

비공개 / NDA

Cypress Hills Partners 인턴십 기간 중 개발한 풀스택 대출 관리 대시보드입니다. 수동 추적을 쿼리 가능하고 CRUD가 활성화된 관리 시스템으로 대체하여 직원들에게 고객 대출 기록 관리를 위한 구조화된 인터페이스를 제공하기 위해 구축되었습니다.

백엔드NestJS
프론트엔드React, Refine
데이터베이스PostgreSQL
APIGraphQL
도구Git, Postman
  • 고객 대출 기록에 대한 전체 CRUD 작업을 갖춘 GraphQL API
  • 정렬 및 필터링 가능한 데이터 테이블을 갖춘 React + Refine 관리 UI
  • 금융 기록 무결성을 위해 설계된 PostgreSQL 스키마
  • 다양한 직원 접근 수준을 위한 역할 인식 뷰
문제 해결 & 배운 점

문제 해결

GraphQL과 Refine 프레임워크 모두 이 프로젝트 시작 시 처음 접하는 기술이었습니다. 빠르게 적응하기 위해 리졸버와 Refine 데이터 제공자가 어떻게 연결되는지 이해하는 소형 프로토타입을 먼저 만든 후, 그 이해를 실제 시스템에 적용했습니다. 이를 통해 통합 오류를 크게 줄일 수 있었습니다.

배운 점

GraphQL이 REST와 쿼리 유연성 측면에서 어떻게 다른지, 그리고 Refine이 일반적인 관리 UI 패턴(CRUD, 페이지네이션, 필터링)을 추상화하여 개발자가 비즈니스 로직에 집중할 수 있게 하는 방법을 배웠습니다. 실제 클라이언트 환경에서 타입화된 NestJS + PostgreSQL 백엔드 작업 경험도 쌓았습니다.

향후 개선 사항

  • 대시보드 수준의 분석 추가 (대출 상태 요약, 연체 알림)
  • 보고용 CSV/PDF 기록 내보내기
  • 모든 기록 변경에 대한 감사 로그 구현
